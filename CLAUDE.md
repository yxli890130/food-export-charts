# 中国食品出口机会探索 · 项目规则

## 产品目标

把贸易数据转化为可验证的“HS6 统计产品 × 目的国”候选，帮助 B2B 外贸团队回答：

1. 某个产品主要出口到哪些国家、出口额是多少；
2. 某个国家主要从中国进口哪些产品、出口额是多少；
3. 某个 HS4 下有哪些更具体的 HS6 统计产品。

网页是研究工具，不是自动推荐器。出口额不等于需求、利润或进入机会；HS6 不等于可销售 SKU。

## 数据口径

- 主贸易统计：UN Comtrade；OEC/BACI 仅用于现有数据迁移和差异复核。
- 报告国：中国（M49 156）；流向：出口；默认年度：2024。
- 计算使用原始美元；人民币仅是显示层，汇率必须记录来源与适用年度。
- 同一图表不混合多个统计来源的金额。
- 缺失值不得当作 0；父子或伙伴加总差异必须记录，不静默修正。
- API Key 只从 `COMTRADE_API_KEY` 环境变量读取，通过请求头传递，不得写入 URL、文件、缓存或日志。

## 目录约定

```text
scripts/                 数据审计、抓取、构建与校验脚本
.cache/trade/            原始 API 缓存（不提交，不发布）
public/data/manifest.json 前端数据清单、来源和完整性
public/data/overview/    首屏与总览数据
public/data/categories/  按 HS2/HS4 分块的产品数据
public/data/countries/   按目的国分块的数据
public/data/products/    按 HS6 分块的目的地数据
public/data/matrix/      产品 × 国家矩阵数据
src/lib/                 类型、加载、计算和格式化
src/components/          页面与可视化组件
docs/dev-log.md          进度、证据、失败命令和下一步
```

生成数据以脚本为唯一真源，不手改 `public/data/` 中的派生文件。临时测试文件放 `.cache/`，任务结束清理无用临时文件。

## 阶段顺序

P0 规则与数据审计 → P1 静态数据构建 → P2 三维探索界面 → P3 初步机会筛选 → P4 验证与交接。每阶段验证通过再进入下一阶段。

## 交互原则

- 默认人民币展示，允许切换美元；排序和指标基于美元。
- 三个一级入口：按产品看国家、按国家看产品、产品 × 国家矩阵。
- 重要数字直接显示，悬浮只补充信息；移动端不依赖悬浮。
- 图表必须提供可读表格；颜色不作为唯一信息载体。
- URL 保存国家、HS 层级、币种和视图，浏览器前进/后退有效。
- 机会初筛使用独立、可解释条件，不输出无证据的综合机会分。

## 完成标准

- 数据文件包含来源 URL、发布者、年度、抓取时间、HS 版本、查询参数、完整性和缺口。
- 父子金额、世界总额与伙伴加总差异有自动测试。
- 核心路径可完成：HS2 → HS4 → HS6 → 目的国，以及目的国 → HS4/HS6。
- 运行 `npm test`、`npm run test:e2e`、`npm run build`，并用浏览器真实操作验证。
- 完工更新 `README.md` 和 `docs/dev-log.md`；不自动 push 或部署。

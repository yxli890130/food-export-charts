# HS4 产品搜索与结果反馈 Implementation Plan

日期：2026-07-21

## 目标

在“按产品看目的国”视图增加可分享、可恢复、可访问的 HS4 产品搜索，解决 116 个产品只能滚动定位的问题。

## 实施顺序

1. 扩展 URL 状态：`ExplorerState.q`、合法化规则、`makeExplorerQuery` 序列化，并先补状态测试。
2. 在 `Explorer` 与 `Analysis` 之间透传 `q`，确保分析视图导航保留搜索词。
3. 在 `ProductsView` 增加本地即时输入、150ms URL 同步、清除按钮、结果数量和无结果状态。
4. 增加安全文本高亮，覆盖 HS 编码、中英文名称。
5. 添加与现有编辑风格一致的响应式 CSS，不引入新依赖。
6. 增加 Playwright 测试：中文/编码搜索、URL 恢复、清除、无结果提示。
7. 修正文档冲突，更新 README、规格和 dev-log。
8. 执行 `npm test`、`npm run test:e2e`、`npm run build` 和浏览器真实走查。

## 验证门

- 搜索不改变金额和排序口径。
- 搜索与 HS2、机会初筛可组合。
- URL 刷新恢复，清除后 `q` 消失。
- 无结果时明确说明仅代表当前数据范围。
- 桌面和移动布局均无溢出。

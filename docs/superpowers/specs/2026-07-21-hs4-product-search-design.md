# HS4 产品搜索与结果反馈设计

日期：2026-07-21  
状态：已实施并验证

## 目标

解决“按产品看目的国”视图中 116 个 HS4 产品只能依靠滚动查找的问题，让用户可以按 HS 编码、中英文名称快速定位产品，同时不改变现有产品→国家分析路径和机会初筛逻辑。

## 用户路径

```text
进入“按产品看目的国”
→ 输入 HS 编码、中文名或英文名（此时列表保持不变）
→ 点击“搜索”或按 Enter 提交
→ 看到匹配数量和筛选后的产品
→ 展开产品查看 TOP15 目的国
→ 清除搜索恢复完整产品列表
```

## 交互规则

- 搜索框放在产品视图标签和机会初筛面板之间，作为高频主操作。
- 支持 HS4 编码（输入 `0712` 或 `HS0712`）、中文名称和英文名称，不区分英文大小写。
- 自动去除关键词首尾空格；空关键词展示当前 HS2 范围内的全部产品。
- 搜索结果继续按 2024 年中国对全球出口额从高到低排列。
- 搜索与已有 HS2 范围、机会初筛条件叠加生效。
- 结果反馈说明“找到 N 个产品”；有其他筛选条件时明确说明结果基于当前范围。
- 无结果时展示可行动提示：检查 HS 编码、尝试更短关键词或清除筛选；不得显示空白列表。
- 输入框为草稿状态，输入字符时不改变产品列表、不改变 URL，避免结果区域高度变化造成页面跳动。
- 点击“搜索”按钮或按 Enter 后才应用关键词、更新结果和 URL。
- 草稿与已应用关键词不一致时，显示“输入已修改，点击搜索或按 Enter 查看结果”。
- 搜索按钮在输入为空或草稿与当前已应用关键词一致时禁用。
- 提供一键清除按钮，清除输入、已应用搜索与 URL 参数，并恢复完整列表。
- 关键词写入 URL 的 `q` 参数；刷新、复制链接和浏览器前进/后退后恢复。
- 在产品卡标题中使用语义化 `<mark>` 标记匹配片段；不使用危险 HTML 注入。

## 状态与数据流

- `ExplorerState` 新增可选 `q?: string`。
- `resolveExplorerState` 只保留 trim 后最多 80 字符的单行关键词；空值、纯空格或含控制字符的异常内容被清理。
- `makeExplorerQuery` 在关键词非空时写入 `q`。
- `Explorer` 将 `q` 传给 `Analysis`，导航时默认保留；切换到国家/矩阵不丢失，但仅产品视图使用。
- `ProductsView` 将输入草稿与已应用关键词分离；只有提交表单或清除搜索时调用 `router.replace`，输入过程不触发路由导航或结果过滤。
- 产品过滤顺序：HS2 范围 → 搜索关键词 → 机会初筛 → 出口额降序。

## 组件边界

- `ProductSearch`：负责搜索输入、清除按钮、结果数量和无结果反馈。
- `HighlightedText`：安全拆分字符串并标记匹配部分。
- 产品过滤保持在 `ProductsView` 的 `useMemo` 中，不引入新的数据文件或依赖。

## 异常与边界

- 搜索只针对当前已发布的 116 个 HS4，不声称覆盖全部食品。
- 英文名缺失时只匹配现有中文名和 HS 编码。
- 无匹配不是“没有该产品”，只表示当前 13 个 HS2、2024 年已发布数据范围内没有匹配。
- 搜索不改变出口金额、排序口径或机会筛选结论。

## 验收标准

1. 输入 `0712`、`HS0712` 或“干制蔬菜”，点击搜索或按 Enter 后均能定位 HS0712。
2. 仅输入但未提交时，产品列表、结果数量和 URL 保持不变。
3. 英文名称支持不区分大小写匹配。
4. URL 包含 `q`，刷新后关键词和结果仍恢复。
5. 清除按钮移除 `q` 并恢复完整产品列表。
6. 搜索可与机会初筛组合使用。
7. 无结果时显示边界说明和下一步操作。
8. 桌面端和移动端输入、搜索、反馈、清除按钮均可用。
9. `npm test`、`npm run test:e2e`、`npm run build` 全部通过。

## 非目标

- 不接入拼音搜索、模糊纠错或搜索建议服务。
- 不搜索 HS6、国家、税则行或外部商品 SKU。
- 不新增数据库、接口或第三方搜索库。
- 不在本轮实现五年趋势、目标国份额或竞争供应国。

# 中国食品出口机会探索

> 🎯 面向外贸从业者的数据探索工具，帮助快速定位中国食品出口的产品-国家组合机会。

---

## 项目背景

做外贸选品时，最核心的三个问题是：
1. **某个产品，主要出口到哪些国家？出口额多少？**
2. **某个国家，主要从中国进口哪些食品？进口额多少？**
3. **哪些产品-国家组合有潜在机会，值得进一步研究？**

公开贸易数据（UN Comtrade、OEC/BACI、海关统计）虽然存在，但多为原始格式，普通人难以直接用于商业分析。这个项目把复杂的海关统计数据转化为直观可操作的探索界面，让非数据专业的外贸从业者也能快速筛选方向。

本项目遵循"**先列证据，再下判断**"的原则：只展示经过验证的出口金额和占比，不输出黑箱"机会评分"，不把出口额直接等同于利润或进入建议，不夸大数据覆盖范围。

## 功能概览

### 📊 核心分析视图

| 视图 | 功能 | 覆盖范围 |
|------|------|----------|
| **按产品看目的国** | 可按 HS4 编码、中英文名称搜索；每个产品展开后显示 TOP15 出口目的国、金额和占比 | 116 个有出口记录的 HS4 产品 |
| **按国家看产品** | 每个目的国展开后显示 TOP30 出口产品、金额和占比 | 198 个国家/地区 |
| **产品 × 国家交叉矩阵** | TOP15 目的国 × 前 60 个 HS4 产品的热力图，单元格可点击跳转 | 60 × 15 = 900 个交叉组合 |
| **HS6 细项下钻** | 从 HS2 → HS4 → HS6 逐级查看更具体的统计品目 | 874 个 HS6 定义 |
| **关税线查看** | 美国 HTS10、日本九位统计品目的官方细分品目 | 两个主要市场 |
| **机会初筛器** | 按最低出口额、覆盖国家数、TOP3 目的国集中度组合筛选候选 | 所有 116 个 HS4 |

### ✨ 交互特性

- **人民币/美元双币种**：汇率采用 2024 年实际平均，计算用原始美元，显示可切换
- **URL 状态保存**：所有视图选择、币种、HS 层级、国家和产品搜索词都保存在 URL 中，可复制分享
- **可访问性设计**：所有图表均有表格降级，键盘可操作，颜色不单独传递信息
- **边界提示**：每页都明确标注数据来源、统计口径、覆盖范围和缺失内容

## 数据来源与口径

| 数据项 | 来源 | 统计年份 | 说明 |
|--------|------|----------|------|
| HS4 产品-国家出口额 | **UN Comtrade** | 2024 | 主统计口径，官方海关数据 |
| HS6 编码与名称 | **UN Comtrade** | 2024 | 6 位编码定义与描述 |
| 美国 HTS10 | **USITC** | 2024 | 美国海关 10 位关税编码 |
| 日本九位码 | **日本财务省关税局** | 2024 | 日本九位统计品目 |
| 汇率 | **国家外汇管理局** | 2024 | 人民币对美元交易日平均汇率 |

### 数据边界与免责声明

> ⚠️ **重要边界**：本工具的结论是"研究起点"，不是"进入建议"
>
> - ✅ 出口额是已发生的历史统计，不等于未来需求、利润或竞争程度
> - ✅ HS6/HTS10/九位码是统计编码，不是可直接销售的 SKU
> - ✅ 不同统计口径（Comtrade vs OEC/BACI）的金额不可直接相加
> - ✅ 数据覆盖 13 个 HS2 食品相关章节，不包含所有食品品类
> - ✅ 未覆盖：准入认证、关税、贸易救济、供应商、买家证据等商业决策必需信息

## 使用方法

### 本地运行

```powershell
# 进入项目目录
cd D:\Codex\food-export-charts

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

打开浏览器访问：**http://localhost:3000**

### 主要入口

| 路径 | 说明 |
|------|------|
| `/` | 主入口，带阶段提示的引导模式 |
| `/explore` | 自由探索模式，无引导 |

### URL 参数示例

```
# 看产品-国家矩阵，人民币
http://localhost:3000/?scope=global&currency=cny&tab=matrix

# 看日本的主要进口产品
http://localhost:3000/?scope=global&currency=cny&tab=countries&country=392

# 展开 HS07 章节的产品列表
http://localhost:3000/?scope=global&currency=cny&tab=products&hs2=07
```

### 重新构建数据（需要 Comtrade API Key）

```powershell
# 设置 API Key（仅存储在环境变量，不写入任何文件）
$env:COMTRADE_API_KEY = "your-api-key-here"

# 运行数据构建脚本
python scripts/build_food_export_data.py

# 同步到 public 目录
npm run sync:data
npm run sync:hs6
npm run sync:markets
npm run sync:tariff-lines
```

### 测试与构建

```powershell
# 单元测试
npm test

# E2E 测试（需要开发服务器运行）
npm run test:e2e

# 生产构建
npm run build
```

### 部署

本项目是标准 Next.js 应用，可直接部署到 **Vercel**：

1. Fork 本仓库到你的 GitHub
2. 在 Vercel 导入项目
3. 无需配置环境变量（数据已预计算为静态 JSON）
4. 点击部署

## 数据验证原则

本项目严格遵循以下数据验证原则：

1. **密钥安全**：API Key 仅从环境变量读取，通过请求头传输，绝不写入 URL、文件、日志或缓存
2. **父子金额校验**：HS2 总额 = 下属所有 HS4 之和，误差超过 0.1% 报警
3. **国家加总校验**：各目的国出口额之和 ≤ 全球总额，差异标注为"其他"
4. **来源透明**：每个数据文件包含来源 URL、抓取时间、参数和统计口径
5. **边界明确**：明确标注"有数据""无数据""数据暂未覆盖"三种状态，不混淆
6. **不推断**：缺失数据不补零、不估算，明确标注"未覆盖"

## 下一步

本项目的长期方向是从"数据看板"进化为"选品辅助工具"，核心演进路径：

1. **趋势层**：加入五年历史数据，识别增长/下降/波动品类
